| Код | Требования                                                                                                      | Комментарий |
|-----|-----------------------------------------------------------------------------------------------------------------|-------------|
| F   | Функциональные (Functionality)                                                                                  |             |
| F1  | Функции умного дома должны быть доступны из текущего приложения владельцев                                      |             | 
| F2  | Владелец долен видеть список устройств умного дома доступных ему                                                |             |
| F3  | Владелец умного дома не должен видеть "чужие" устройства умного дома                                            |             |
| F4  | Владелец умного дома долен иметь возможность добавлять биометрическую информацию для устройств умного дома      |             |
| F5  | Владелец умного дома должен иметь возможность добавлять номера автомобилей в базу данных устройств умного дома  |             |
| F6  | Владелец должен иметь возможность установить время доступа для гостей и автомобилей                             |             |
| F7  | Все факты успешного или не успешного доступа через устройства умного дома должны фиксироваться в системе        |             |
| F8  | Владелец умного дома должен иметь возможность отзывать права доступа для гостей                                 |             |
|     |                                                                                                                 |             |
| U   | Удобство использования (Usability)                                                                              |             |
| U1  | Функции умного дома должны предоставляться в привычном для владельцев интерфейсе                                |             |
| U1  | О факте успешного доступа гостей должено направляться PUSH уведомление владельцу добавившего гостя              |             |
|     |                                                                                                                 |             |
| R   | Надёжность (Reliability)                                                                                        |             |
| R1  | 24/7 и доступность 99.9% для системы умного дома                                                                |             |
| R2  | Все межсервисные взаимодействия должны защищаться TLS не ниже версии 1.2                                        |             |
| R3  | Подключение к внешнему сервису умного дома должно проверяться на уровне mTLS                                    |             |
| R4  | Провести разработку ролевой модели для приложения сервисов ЖКХ и разрабатываемого микросервиса                  |             |
| R5  | Менеджеры внешней системы умного дома не должны иметь доступ к внутреннему микросервису умного дома             |             |
| R6  | Доступ к биометрической информации должен быть только у владельца его добавившего                               |             |
| R7  | При необходимости получения доступа к биометрической информации права должны выдаваться только на время работ   |             |
|     |                                                                                                                 |             |
| P   | Производительность (Performance)                                                                                |             |
| P1  | Время срабатывания устройств умного дома должно быть не более 3 сек.                                            |             |
|     |                                                                                                                 |             |
| S   | Поддерживаемость (Supportability)                                                                               |             |
| S1  | При разработке микросервисов выполнить документирование и согласование прав доступа к биометрической информации |             |
| S2  | При разработке провести документирование методов авторизации владельцев и менеджеров ЖКХ                        | техдолг     |
| S3  | Документирвоание должно выполняться в формате докментация как код                                               |             |
|     |                                                                                                                 |             |
|     |                                                                                                                 |             |
| +R  | + Ограничения (Restrictions)                                                                                    |             |
| +R1 | При разработке новых микросервисов использовать БД Postgre                                                      |             |
| +R2 | При разработке новых микросервисов использовать SpringBoot                                                      |             |
|     |                                                                                                                 |             |
